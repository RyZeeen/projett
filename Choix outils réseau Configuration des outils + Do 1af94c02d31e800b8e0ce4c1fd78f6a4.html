Choix outils réseau / Configuration des outils + Documentation

1. Routeur & SD-WAN

Choix : Cisco Catalyst 8000 Series (ex : C8500-12X)

Justification :

  • SD-WAN natif : Permet une gestion intelligente du trafic et une connectivité multi-sites sécurisée.
  • Haute disponibilité : Supporte des protocoles comme VRRP/HSRP pour la redondance.
  • Sécurité avancée : Inspection du trafic, pare-feu et protection contre les menaces.
  • Performance : Débit suffisant pour supporter des connexions 10G+ avec routage optimisé.

2. Switchs (Core & Access)

Choix pour le cœur de réseau : Cisco Catalyst 9500 Series

Justification :

  • Capacité 10G/40G/100G pour répondre aux besoins des clusters Kubernetes.
  • Fonctions avancées de VLAN et VXLAN pour une segmentation flexible du réseau.
  • Haute disponibilité avec stacking et redondance.

Choix pour l'accès : Cisco Catalyst 9300

Justification :

  • Support des VLANs pour isoler les différents services et équipes.
  • PoE+ (Power over Ethernet) pour alimenter les AP Wi-Fi et autres équipements.
  • Gestion centralisée avec SDN (ex : Cisco DNA Center, Aruba Central).

3. Pare-feu & Sécurité

Choix : Palo Alto PA-3220

Justification :

  • Filtrage avancé du trafic avec IDS/IPS et contrôle applicatif.
  • Intégration avec SD-WAN pour une sécurité cohérente sur l’ensemble de l’infrastructure.
  • Zero Trust Network Access (ZTNA) pour renforcer la sécurité sur les clusters Kubernetes.

4. Wi-Fi & Réseau sans fil

Choix : Cisco Meraki MR56

Justification :

  • Wi-Fi 6E pour une meilleure densité et latence réduite.
  • Gestion centralisée cloud-native avec Meraki ou Aruba Central.
  • Segmentation et sécurité avec SSID dédiés et politiques VLAN.

5. Load Balancer pour Kubernetes

Choix : HAProxy

Justification :

  • Distribution intelligente du trafic entre les nœuds Kubernetes.
  • SSL/TLS offloading pour réduire la charge sur les serveurs applicatifs.
  • Scalabilité et haute disponibilité pour les services critiques.

6. Orchestration & Monitoring

  • Monitoring Réseau : Zabbix, PRTG, ou Cisco DNA Assurance.
  • Observabilité Kubernetes : Prometheus + Grafana.
  • Gestion des logs : Elastic Stack (ELK) ou Splunk.

    • Création du profil IKE (défini comment le pare-feu établit une connexion sécurisée avec un pair VPN)

set network ike crypto-profiles ike-crypto-profiles "SDWAN-IKE" lifetime seconds 28800
    • Création du profil IPsec
      set network ipsec crypto-profiles ipsec-crypto-profiles "SDWAN-IPsec" esp authentication sha256
set network ipsec crypto-profiles ipsec-crypto-profiles "SDWAN-IPsec" esp encryption aes-256-cbc
set network ipsec crypto-profiles ipsec-crypto-profiles "SDWAN-IPsec" lifetime seconds 3600

  • Configuration du switch Genève
    !------------------------------------------------------------
! Configuration de base du Catalyst 9500
!------------------------------------------------------------

! 0. Activation du routage IP (nécessaire pour l'inter-VLAN)
ip routing

!------------------------------------------------------------
! 1. Création des VLANs
vlan 10
 name SOC_SERVERS
!
vlan 20
 name EQUIPE_SUBNET
!

!------------------------------------------------------------
! 2. Configuration des interfaces VLAN (SVI)
interface Vlan10
 description "SVI VLAN 10 - Serveurs SOC"
 ip address 172.16.10.1 255.255.255.0
 no shutdown
!
interface Vlan20
 description "SVI VLAN 20 - Équipe Subnet"
 ip address 172.16.20.1 255.255.255.0
 no shutdown
!

!------------------------------------------------------------
! 3. Routage inter-VLAN (automatique via ip routing)
! Aucune route statique nécessaire ici pour l’inter-VLAN
! Les VLANs sont directement connectés au routeur L3

! (Optionnel) Exemple d’ACL pour restreindre les flux de VLAN 20 vers VLAN 10
ip access-list extended VLAN20_TO_VLAN10_RESTRICT
 deny ip 172.16.20.0 0.0.0.255 172.16.10.0 0.0.0.255
 permit ip any any
!
interface Vlan20
 ip access-group VLAN20_TO_VLAN10_RESTRICT in
!

!------------------------------------------------------------
! 4. Configuration des interfaces physiques

! 4.1. Trunk vers le Pare-feu
interface GigabitEthernet1/1/1
 description "Trunk vers Pare-feu"
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan 10,20
!

! 4.2. Port pour la borne Wi‑Fi Cisco (VLAN 20)
interface GigabitEthernet1/1/2
 description "Port d'accès pour Borne Wi‑Fi Cisco"
 switchport mode access
 switchport access vlan 20
!

! 4.3. Port pour un serveur SOC (VLAN 10)
interface GigabitEthernet1/1/3
 description "Port d'accès pour Serveur SOC"
 switchport mode access
 switchport access vlan 10
!

!------------------------------------------------------------
! 5. (Optionnel) Port uplink vers le coeur de réseau ou Internet
interface GigabitEthernet1/1/48
 description "Uplink vers cœur de réseau"
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan 10,20
!

!------------------------------------------------------------
! 6. Remarques complémentaires
! - Le switch agit ici comme un routeur Layer 3 pour le trafic inter-VLAN
! - Aucune extension L2 n’est faite entre les sites (chaque site a ses propres VLANs et plages IP)
! - L’ACL utilisée est optionnelle pour la sécurité (blocage des accès de VLAN 20 vers VLAN 10)
!
! Fin de configuration
!------------------------------------------------------------
