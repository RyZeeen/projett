Documentation des outils de surveillance des vulnérabilités (Snyk, Checkmarx).

🧱 1️⃣ Contexte & objectifs

ÉlémentDétail
InfrastructureParis & Genève (serveurs physiques), SD-WAN, cloud Azure & AWS
ApplicationsApplications internes + applicatifs déployés sur cloud
ObjectifIdentifier & corriger les vulnérabilités dans le code, les dépendances, les conteneurs et l'infrastructure

🛠️ 2️⃣ Outils sélectionnés

OutilFonction principale
SnykScan du code source, dépendances, conteneurs & IaC (Infrastructure as Code)
CheckmarxAnalyse SAST (Static Application Security Testing) + complément DAST possible
Complémentaire :- Trivy (open source rapide pour Docker, IaC)- OWASP ZAP (tests dynamiques basiques)

📈 3️⃣ Zones de surveillance

CibleOutil(s)Méthodologie
📦 Code source (GitLab/GitHub)Snyk + CheckmarxAnalyse continue (push/merge)
🐳 Conteneurs Docker (Azure/AWS)Snyk Container / TrivyScan pipeline CI/CD + scan runtime
⚙️ Infra as Code (Terraform, ARM)Snyk IaC / Checkmarx IaCScan à chaque commit + pré-prod
🖥️ VM & OS (Paris/Genève + Cloud)Trivy / SnykScan périodique des images systèmes
🌐 Applications web déployéesCheckmarx DAST / OWASP ZAPTests dynamiques post-déploiement

🔗 4️⃣ Intégration & automatisation

a. 🔄 Pipelines CI/CD (GitLab / GitHub)

  • Snyk :
    • Plugin Snyk dans le pipeline (ex: .gitlab-ci.yml)
    • Scan au moment des merge requests & builds
    • Blocage automatique si vulnérabilités critiques
  • Checkmarx :
    • Intégration webhook : déclenchement analyse SAST après chaque push
    • Rapport directement disponible dans l’interface GitLab/GitHub
    • Export vers SIEM si critique

b. ☁️ Cloud & VM

  • Snyk CLI / API : planification via cron/Ansible pour les VMs locales & cloud
  • Checkmarx : intégration avec Azure DevOps / AWS CodePipeline

c. 🔔 Alerte & ticketing

  • Slack / Teams webhook : notifications en cas de vulnérabilité critique
  • ServiceNow/Jira : création auto de ticket à partir d’un rapport de vulnérabilité
  • SIEM (ex : Splunk) : centralisation des vulnérabilités pour corrélation

🚨 6️⃣ Politique d’alerte & gestion des vulnérabilités

GravitéAction
🔴 CritiqueBlocage pipeline / patch immédiat
🟠 HauteCorrection sous 15 jours
🟡 MoyenneCorrection sous 30 jours
🟢 Faible / InfoSuivi trimestriel
  • Revue mensuelle avec équipe de dev + sécurité
  • Vulnérabilités résiduelles documentées + justifiées (ex: false positive, exception temporaire)

🚀 Exemple de workflow : Déploiement d’une nouvelle feature

1️⃣ Développeur push le code → pipeline GitLab s'exécute

2️⃣ Snyk & Checkmarx analysent automatiquement

3️⃣ Vulnérabilité critique détectée → pipeline bloqué, ticket auto dans Jira

4️⃣ Notification Slack envoyée à l’équipe dev + sécurité

5️⃣ Correction réalisée → nouveau push → pipeline passe OK

6️⃣ Déploiement autorisé après validation

7️⃣ Rapport mensuel mis à jour
